关于React的思考

声明式开发

减少大量dom操作代码量

可以和其他框架并存

react只去管理id=”root”的div的渲染。jQuery可以管理其他div操作。

组件化

  • 区分组件和标签
    xxx
  • 组件之间通信：
    • 父向子：属性
    • 子向父：调用父组件的方法

单向数据流

父组件可以向子组件传值（只读），子组件不能去改变这个值。

why单项数据流：防止其中一个改变时出现bug，难以定位是哪一个组件改变导致的错误。

视图层框架

只解决数据和渲染的问题，不解决react复杂传值问题，可以借助其他redux等数据传递框架

函数式编程

代码逻辑清晰：拆分函数，各司其职；

前端自动化测试便捷：只需要给函数一个数值，查看输出是否符合预期。
